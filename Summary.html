Real-Time Transport Engine Summary

This implementation handles real-time packet transport over a simulated hostile network using a combination of proactive Forward Error Correction (FEC) and an asynchronous jitter buffer.

Architecture Blueprint

Harness Source ➔ Sender ➔ Relay (hostile) ➔ Receiver (Buffer) ➔ Harness Player

Core Engineering Decisions

  • Dual-Payload Packet Bundling: To stay safely under the 2.0x bandwidth overhead cap while completely mitigating single packet drops, each UDP packet contains the current frame and the immediate predecessor.
  • Zero-Blocking Playout Clock: Playout scheduling is isolated on a dedicated POSIX thread to ensure high-resolution 20 ms timing. This guarantees that delayed or reordered network packets never block or stall the playback thread.
  • Low Overhead: This protocol maintains a highly efficient transmission footprint of around ~1.1x, providing a secure margin below the 2.0x system cap.
